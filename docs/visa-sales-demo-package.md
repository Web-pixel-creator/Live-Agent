# Visa Relocation Demo Package

This package is the short commercial demo for the current product wedge:

- visa agencies
- relocation agencies
- cross-border intake teams

The product should be presented as `AI Action Desk`, not as a generic multimodal lab.

## One-Sentence Offer

`AI Action Desk` is an operator-safe agent for visa and relocation teams that qualifies leads, books consultations, chases missing documents, and prepares CRM or handoff actions from one live workspace.

## Who This Is For

Use this package when the audience is:

1. a visa or relocation agency founder,
2. an operations lead,
3. a sales lead,
4. a customer support manager,
5. a partner evaluating pilot automation.

Do not lead with `Storyteller` for this audience. Keep `Simulation Lab` as a secondary proof that the platform can rehearse and train scenarios.

## Main Business Problem

These teams usually lose time and money in four places:

1. first-response qualification is repetitive,
2. consultation booking turns into back-and-forth chat,
3. missing documents require repeated follow-ups,
4. final updates still have to be copied into CRM or handed off to a human.

This demo package should show that the agent does real work across that chain.

## Demo Flow

Run the demo in this order:

1. `Run Visa Intake Demo`
2. `Review Visa Draft Result`
3. `Run Missing Docs Follow-up`
4. `Review Follow-up Result`
5. `Run Consultation Reminder`
6. `Review Reminder Result`
7. `Run CRM Update Demo`
8. `Review CRM Handoff Result`
9. `Run Case Escalation`
10. `Review Human Handoff Result`

Use `Reset Visa Demo` between takes when you need a clean starting point.

## What Each Step Proves

### 1. Visa Intake

Shows that the agent can:

- understand the lead,
- structure intake data,
- prepare a safe UI draft,
- stop at the approval boundary.

### 2. Missing Documents Follow-up

Shows that the agent can:

- detect what is missing,
- generate a useful follow-up,
- keep the case moving without a human writing every message.

### 3. Consultation Reminder

Shows that the agent can:

- keep the booked consultation alive,
- remind the client what to prepare,
- maintain context from the same case.

### 4. CRM Update

Shows that the agent can:

- write back a structured handoff packet,
- prepare the system update,
- leave the operator with a clean next step.

### 5. Case Escalation

Shows that the agent can:

- recognize a case that needs a human,
- route it to the right owner,
- preserve control and proof at the approval boundary,
- produce a final handoff summary.

1. the product is not a chatbot,
2. the product is not only a browser runner,
3. the product helps a real service team move one client case from first contact to human handoff.

## Common Buyer Questions

### What does this replace?

It removes repetitive first-response work, manual reminders, repetitive document chasing, and part of the CRM update process.

### Does it act without control?

No. The product is designed around approval boundaries, verified summaries, and operator-visible state.

### Is this only a demo?

The demo uses deterministic fixtures for repeatability, but it mirrors the same runtime shape used for live UI tasks and operator approvals.

### Why not just use chat?

Chat stops at advice. This product moves the case forward and prepares the next system action.
